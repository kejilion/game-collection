# 🧗 敲冰块大逃杀 · Ice-Climber Arena

一个**多人在线**的「敲冰块（Ice Climber）+ 爬塔求生逃杀」网页游戏。
一个 URL 连接所有浏览器客户端，由 **权威服务器（authoritative server）** 统一模拟，
未来可用 Docker 一键部署到服务器。

> 2D 卡通独立游戏质感 · 全部美术由代码程序化绘制（无外部素材）· 服务器权威 + 客户端预测的实时联机架构。

---

## ✨ 玩法概览

- **起名 + 捏人**：输入名字，选择帽子样式、肤色、**服饰颜色**、帽子颜色，人物形象由程序实时绘制预览。
- **向上攀塔逃生**：10 层平台，**越往上行动区域越窄**。跳跃在层与层之间穿梭。
- **砖块系统**：
  - 🧊 **可击碎砖块**：跳起用头顶碎，碎裂后掉落（有概率掉道具）。
  - 🪨 **不可击碎砖块**：坚固岩石。
  - ↔️ **移动砖块**：左右往复的升降平台，可站上去搭便车。
  - 💨 **加速砖块**：光滑冰面，移动更快、更滑。
- **危险层**：部分楼层会**结冰、冰锥从天而降**，砸中扣血。
- **小怪（共 6 种，每关随机取 2~3 种）**：`冰拳企鹅`贴身追击并**冰冻（控制）**你、`寒冰法师`远程瞄准冰弹、`冲锋兽`蓄力后高速突进、`跳跳雪怪`蹦跳逼近难以命中、`冰甲重兽`血厚体大击退凶猛、`霜吐者`三连扇形吐弹。它们的**巡逻范围已大幅扩大**，会在整层来回游走、更远就开始锁定追杀。
- **BOSS（共 6 种，每局随机出没 1 只，盘踞高层）**：`冰霜巨人`(近战巨力击退)、`暴风雪法师`(密集扇形弹幕)、`寒冰女王`(召唤小怪助阵)、`冰川猛犸`(全层来回猛冲)、`霜翼飞龙`(悬空盘旋·空降冰刺)、`冰晶魔像`(环形弹幕爆发)。开局会广播 BOSS 名号与所在楼层。
- **道具（拾取即用，共 5 种）**：🔥 火球（普攻变发射火球）、❤️ 回血、⬆️ 提升跳跃高度、💨 加速（移速提升，限时）、🛡️ 免伤（护盾无敌泡，限时——免疫一切伤害与冰冻）。
- **战斗**：玩家只有普攻、**攻击力极低**，可以互相攻击但**杀死对方很慢**。
- **坠落伤害**：下落**超过 2 层**才扣血，跨度越大摔得越惨。
- **死亡**：血量归零 → **回到第一层**重新爬。
- **胜利**：爬到最顶层，**救援飞机出现**，**跳起来**被飞机接走即逃生成功，记录名次与用时。
  **前三名全部被接走 → 本局结束**，全员进入下一局，并提示「开服后第几局」。

## 🎮 操作

| 按键 | 作用 |
| --- | --- |
| `←` `→` | 左右移动 |
| `↑` / `空格` | 跳跃 |
| `A` | 攻击 |

界面 HUD：左上显示**局数 / 楼层 / 血量 / 增益**，顶部中间显示**开局计时**，
右上显示**实时排行**与一个迷你塔，标出所有玩家所处楼层与位置。

## 🏆 持久化排行榜

每次有人被飞机接走都会写入 `server/data/leaderboard.json`：**按逃生用时排名**，
并记录该纪录是在**开服后第几局**创造的。重启 / Docker 重新部署后数据保留（挂载 volume）。

## 🚀 运行

### 本地开发

```bash
npm install
npm start            # 默认 http://localhost:3000
# 或指定端口：
node server/index.js 8080      # 或  PORT=8080 npm start
```

浏览器打开地址即可，多开几个标签页 / 让朋友访问同一地址即可联机。

```bash
npm test             # 运行物理 / 关卡 / 排行榜 / 房间模拟的单元测试
npm run dev          # node --watch 热重载
```

### Docker 部署

```bash
docker compose up -d            # 构建并后台启动，映射 3000 端口
# 访问 http://<服务器IP>:3000
```

`docker-compose.yml` 已把 `./server/data` 挂载为卷以持久化排行榜。
反向代理（Nginx/Caddy）记得放行 **WebSocket** 升级头（Socket.IO 需要）。

## 🧱 架构

参考 `kejilion/game-collection` 中 `arena-brawl` 的组织方式（`public/` 客户端 + `server/` 后端 +
Docker），但因为是实时多人，采用**服务器权威**的网络模型：

```
敲冰块2/
├── server/                     # Node 后端（权威模拟）
│   ├── index.js                # Express 静态托管 + Socket.IO 网关 + REST 接口
│   ├── game/
│   │   ├── GameRoom.js         # 60Hz 主循环：模拟/战斗/回合/胜负/快照广播
│   │   ├── Level.js            # 程序化关卡生成（10 层塔，保证可攀爬主路径）
│   │   ├── Player.js           # 玩家实体与状态序列化
│   │   ├── Monster.js          # 怪物 AI：6 种小怪 + 6 种 BOSS（追击/突进/蹦跳/远程/召唤/弹幕/悬空），数据表驱动
│   │   ├── entities.js         # 道具 / 抛射物 / 冰锥
│   │   └── Leaderboard.js      # JSON 持久化历史排行
│   └── data/                   # 运行时写入 leaderboard.json（挂载为卷）
├── public/                     # 浏览器客户端
│   ├── index.html  css/        # 菜单 + 捏人 + HUD + 结算覆盖层
│   ├── shared/                 # ★ 客户端与服务器共享的代码
│   │   ├── constants.js        #   全部数值调参 / 枚举 / 世界几何
│   │   └── physics.js          #   确定性物理（同一份代码两端运行）
│   └── js/
│       ├── main.js             # 启动 + 游戏主循环 + 状态机
│       ├── network.js          # Socket.IO 封装 + 时钟同步 + 远端插值
│       ├── prediction.js       # 本地玩家预测 + 快照协调（reconcile）
│       ├── input.js            # 键盘输入
│       ├── renderer.js         # Canvas 渲染 + 粒子特效
│       ├── characters.js       # 程序化卡通人物绘制（捏人与游戏内共用）
│       └── ui.js               # 捏人 / 排行榜 / 时间格式化
├── test/                       # node:test 单元 + 模拟测试
├── Dockerfile  docker-compose.yml  .dockerignore
└── package.json
```

### 网络模型（为什么这样设计）

- **服务器权威**：所有物理、战斗、怪物、道具、回合都在服务器以固定 **60Hz** 步进，
  每 **30Hz** 向所有客户端广播一份世界快照（`io.volatile` 丢弃过期包）。防作弊、保证一致。
- **客户端预测**：本地玩家用 **同一份** `shared/physics.js` 即时模拟自己的输入，
  手感无延迟；收到权威快照后做**软协调**（误差大则瞬移校正，如复活/被接走）。
- **远端插值**：其他玩家 / 怪物 / 抛射物按 ~100ms 延迟在两帧快照间插值，平滑流畅。
- **移动平台是时间的纯函数**：`x = base + sin(t·speed + phase)·range`，两端用同步时钟各自计算，
  无需为平台位置额外发包。
- **时钟同步**：客户端定期 `timesync` 估算与服务器的时间偏移（EMA 平滑）。

### 美术

人物、怪物、平台、飞机、道具、雪山背景**全部由 Canvas 形状程序化绘制**，
统一的「**霜冻雪山卡通**」风格——圆润、明亮、可爱，未模仿任何参考游戏的美术。

## 🔧 主要调参

数值集中在 [`public/shared/constants.js`](public/shared/constants.js)，改一处两端同时生效：
楼层数/间距、跳跃高度、坠落伤害阈值、攻击力与冷却、怪物/道具/冰锥强度、救援名额、回合时长等。
