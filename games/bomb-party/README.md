# 💣 炸弹派对 Bomb Party

Q 版卡通多人在线炸弹人（泡泡堂 / 经典炸弹人玩法）· **常驻大世界**。**一个房间，一个网址**——所有浏览器打开同一 URL 即进入同一战场，服务端权威模拟，无需注册，开服永久运行。

## 玩法特色

- 🗺️ **常驻大世界**：41×33 大地图，摄像机跟随视野铺满全屏，左上角小地图纵览全局
- 🎮 **经典炸弹人对战**：放炸弹、炸砖块、十字火焰、连锁引爆；随机空旷处出生，死亡 2.5 秒自动重生
- ⚔️ **点数竞技**：炸砖 +10 · 怪物 +50~150 · 击杀玩家 +300（连杀有加成）· 金史莱姆 +500；实时排行 + 历史最佳双榜（Tab 查看）
- 👾 **四种怪物**：史莱姆（乱走）、幽灵（追人）、小恶魔（高速）、✨金史莱姆（无害会逃、抓到大赏，全场限一只）
- 🎁 **砖块藏道具**：💣 炸弹数 / 🔥 火力 / ⚡ 移速 / 🛡️ 护盾；死亡会掉落生前强化，道具过期消失
- 🧱 **砖块再生**：被炸的砖块 45~90 秒后重新长出，世界资源永续循环
- 🎭 **选角界面**：8 种颜色任选 + 昵称；⚙️ 菜单支持继续 / 音乐音效开关 / 回到选角 / 退出
- 📶 **高延迟优化**：本机移动本地预测 + 平滑纠偏（零延迟手感），其余实体自适应插值，音效按距离衰减，实时 ping 显示
- 📱 手机适配：左半屏任意位置滑动移动（动态摇杆）+ 右下炸弹按钮
- 🏆 排行榜持久化（历史最高分 / 累计击杀），重启容器不丢失
- 🔊 全程序化 Q 版画面与 WebAudio 合成音效，零图片/音频资源

## Docker 部署（推荐）

```bash
docker compose up -d --build
# 或
sh scripts/deploy.sh
```

打开 `http://服务器IP:3000`，把网址发给朋友即可联机。

- 端口：修改 `docker-compose.yml` 中的 `ports`（如 `"3001:3000"`）
- 数据：排行榜存于 `/opt/bomb-party/data`（宿主机），按需修改挂载路径
- 健康检查：`GET /health`

## 本地开发

```bash
npm install
npm start        # http://localhost:3000
npm test         # 世界模拟单元测试（node --test）
```

### 网络与延迟设计

- **协议**：WebSocket JSON。客户端只发输入（方向 / 放炸弹），服务端 30Hz 权威模拟（时间累积器对齐墙钟），15Hz 广播快照，杜绝作弊。
- **本机玩家**：客户端用与服务端一致的碰撞规则做本地预测，快照到达后小步纠偏（偏差 >2 格才硬拉回），按键手感零延迟。
- **其他实体**：双快照插值回放，插值缓冲按实测快照间隔与抖动自适应（60~280ms），高抖动网络下依然平滑。
- **事件跨帧累积**：模拟帧之间的所有事件随下一个快照送达，不丢音效/特效。
