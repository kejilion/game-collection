# Audio assets

Procedurally synthesized placeholders generated via `ffmpeg` sine + tremolo filters, in the same spirit as the project's "procedural cartoon art" renderer. Total: 784 KB (well under the 1.5 MB budget).

These are intentionally simple placeholders. For production, regenerate each file using an AI audio service (ElevenLabs Sound Effects, Stable Audio, Suno, etc.) using the prompts below. The procedural files already implement the correct envelope, frequency, and duration for each cue, so swapping in AI-generated WAVs/MP3s requires no code changes — only update the file in place.

## Regeneration recipe (procedural)

```bash
# BGM loop (60 s, layered)
ffmpeg -y \
  -f lavfi -i "sine=frequency=220:duration=60" \
  -f lavfi -i "sine=frequency=330:duration=60" \
  -f lavfi -i "sine=frequency=440:duration=60" \
  -filter_complex "[0:a]volume=0.25,tremolo=f=4:d=0.6[a0];[1:a]volume=0.20,tremolo=f=6:d=0.5[a1];[2:a]volume=0.15,tremolo=f=8:d=0.4[a2];[a0][a1][a2]amix=inputs=3:duration=longest,afade=t=in:st=0:d=1.5,afade=t=out:st=58.5:d=1.5" \
  -ar 44100 -ac 2 -b:a 96k music/bgm-loop.mp3

# SFX — example for swing
ffmpeg -y -f lavfi -i "sine=frequency=400:duration=0.18" \
  -af "afade=t=in:st=0:d=0.01,afade=t=out:st=0.13:d=0.05,tremolo=f=12:d=0.4" \
  -ar 22050 -ac 1 -b:a 64k sfx/swing.mp3
```

## Format constraints

- SFX: mp3, 64 kbps, mono, 22050 Hz, ≤ 1.5 s each.
- BGM: mp3, 96 kbps, stereo, 44100 Hz, must loop cleanly.
- After regenerating audio assets, bump the `ASSET_VERSION` constant in `public/js/audio.js` and the `?v=` query string on its `<script>` tag in `public/index.html` to invalidate cached copies.
